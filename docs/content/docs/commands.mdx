## `yantr init`

Initializes YantrJS in a new or existing project.

### Usage

```bash
yantr init [options]
```

### Options

| Flag | Description |
|------|-------------|
| `--yes` | Use defaults for all prompts |

### What it does

1. **Detects or creates** `package.json` with your chosen framework
2. **Prompts for framework** - Express, Hono, or Fastify
3. **Creates** `yantr.json` configuration
4. **Installs** framework-specific base templates (error-handler, zod-middleware)
5. **Installs** `zod` for validation

### Framework Detection

If your `package.json` already has a framework installed, YantrJS will automatically detect it:

```bash

# Has hono in dependencies â†’ detects Hono  
# Has fastify in dependencies â†’ detects Fastify
```

## `yantr add`

Downloads a component from the YantrJS registry.

### Usage

```bash
yantr add <component> [options]
```

### Options

| Flag | Description |
|------|-------------|
| `--overwrite` | Overwrite existing files |

### Available Components

| Component | Description |
|-----------|-------------|
| `auth` | JWT authentication with refresh tokens |
| `logger` | Pino-based structured logging |
| `database` | Database setup with ORM selection |
| `security` | Rate limiting + Helmet headers |

### Database Selection

When adding `database`, you'll be prompted to choose:

**Database Type:**
- PostgreSQL
- MongoDB

**ORM (for PostgreSQL):**
- Prisma (recommended)
- Drizzle

**ORM (for MongoDB):**
- Mongoose

```bash
yantr add database
# â†’ Select database: PostgreSQL
# â†’ Select ORM: Prisma
# â†’ Creates lib/yantr/database/client.ts, db.ts
# â†’ Installs @prisma/client, prisma
```

## `yantr generate` (Alias: `g`)

Scaffolds boilerplate code for common patterns.

### Usage

```bash
yantr generate <type> <name>
```

### Supported Types

| Type | Description |
|------|-------------|
| `route` | CRUD route with controller and service |

### Example

```bash
yantr generate route users
```

Creates:
- `routes/users.routes.ts` - Route definitions
- `controllers/users.controller.ts` - Request handlers
- `services/users.service.ts` - Business logic

## Configuration (`yantr.json`)

After initialization, your `yantr.json` looks like:

```json
{
  "$schema": "...",
  "projectName": "my-app",
  "srcDir": "./src",
  "framework": "hono",
  "database": {
    "type": "postgres",
    "orm": "prisma"
  },
  "packageManager": "pnpm",
  "installedComponents": ["base", "database", "auth"]
}
```

| Field | Description |
|-------|-------------|
| `framework` | express, hono, or fastify |
| `database` | Database type and ORM (if installed) |
| `installedComponents` | List of installed components |
